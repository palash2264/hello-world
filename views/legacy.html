Responsive Design

There are 3 breakpoints for small, medium, and large screens. There are a variety of helper variables and classes in the styleguide, to easily define style differences for these 3 form factors. Reference the helper at the bottom of the screen to know what form factor your current screen falls under.

Small

0 to $small-screen-max (414px)

Medium

$medium-screen-min (415px) to $medium-screen-max (815px)

Large

$large-screen-min (816px) to $large-screen-max (1050px)

Using Breakpoints

You should write your CSS in a mobile first structure. This means all styles by default will apply to the mobile form factor. The media tags, $medium-screen and $large-screen, can be used to alter styles for additional modifications for the medium and large form factors. We use the Neat library for breakpoint specific code. Check out the documentation there for more detailed information. 

                .responsive-example {
                  font-size: 12px;
                  color: $light-grey;

                  @include media($medium-screen) {
                    font-size: 20px;
                    color: $dark-grey;
                  }

                  @include media($large-screen) {
                    font-size: 32px;
                    color: $deep-blue;
                  }
                }

Responsive Helper Classes

You can use the mobile-only and desktop-only classes to easily designate elements that should be shown only on certain form factors.

Note: You might notice there are only two. Currently, mobile corresponds with the medium and small breakpoints, and desktop corresponds with the large form factor. This is for convenience, as the small and medium forms share the same designs.

Spacing

There are several varibles that can help you more effectively choose spacing for your feature specific styling. Hover over an element to see the spacing highlighted.

$small-spacing represents the smallest amount of spacing between two elements. The small space between a label and the field, or the spacing between field and it's note is an example of small spacing.
$medium-spacing represents the margin along the sides of the page. That's why the variable $gutter-spacing can also be used to refrence this spacing value.

$large-spacing is often used for vertical space between two elements on a page, such as an h2 and the content beneath it.

It's also used to seperate paragraphs from each other.

That's why you can also use the variable $paragraph-spacing.

Collapsable Tables

Collapsable tables will condense into a single-column list with the headings hidden on small form-factors. On large form factors, they will look identical to normal tables.

If you want a table cell to remain a column in all form factors. Add the class keep-column to the td element. Note: this will only work correctly for columns on the far left or far right of the table.

Feature Layout

Features that take advatage of the feature layout classes should be wrapped by an element with the class kp-feature

The feature layout is composed of of 4 sections:

The feature-header represents the feature title (and icon), shown below the KP header.

The feature-menu represents the main links associated with a feature. On mobile, it will be visible on the landing page, and hidden on every other screen. On desktop, it will be represented as a side panel of navigation. Add the active class to the a element representing the active route.

The feature-description represents a short description of the feature. This is only shown on the mobile landing page.

The feature-content should wrap any other content on the page relating to the feature.

Feature Alerts

If you need an alert for the entire screen, put it in an element with the feature-alert class in the top of your .feature-content element. This will pull the notification up, and push the content down appropriately to highlight the alert.

Use the active class to make the notification visible.
