About

The Web Design Styleguide (or as we call it, the styleguide) is both a styleguide document, which you can see here on this site, and a library of styling assets, like images, fonts, and stylesheets. This document was built with the same assets that are imported into responsive KP applications.

If you're interested in learning more about this concept, listen to this talk on Styleguide Driven Development by Nicole Sullivan.

Javascript UI elements

We have added a new page for Javascript UI elements, these are patterns that require the use of Javascript for specific behavior, for example, accordions and custom select menus.

Installation

The styleguide is a NPM module. You can add it as a dependency to your project by running the following command in your project directory: 

npm install -D git+ssh://git@stash.kp.org/rwd/styleguide.git

Then, include the styles in your sass file: 

@import "styleguide/modern"

Since the styleguide contains a variety of front-end assets, which all need to be handled the right way, we encourage you to use the KP Build Plugin. If you are using the Build Plugin, you're all set!

For discussion and help around including Styleguide for AEM integration, log into HipChat (with SSO credentials) and join RWD Styleguide or RWD Build Plugin.

Responsive

For ease of development the following SASS variables are provided that contain defined breakpoints.

Usage

If you want to apply a specific style to let's say the $mobile view try this: 

.element {
  @include media($mobile) {
    background-color: black;
  }
}
Here's another example: if I want to apply $desktop-large with display but hide the view in $device: 

.element{
  @include media($desktop-large) {
    display:block;
  }
  @include media($device){
    display:none;
  }
}

Colors

Include the colors below by using the following Sass function. Pass in one of the color names below as a parameter to get its calculated hex value.

For example:


.element {
  background-color: color("robin-egg");
}

Some colors, like interactive-blue, have aliases defined as variables ($link-color) that can be used on their own without calling the colors function.

Spacing

There are several variables that can help you more effectively choose spacing for your feature-specific styling. Hover over an element to see the spacing highlighted.

Extra tiny spacing

example

$extra-tiny-spacing

Tiny Spacing

example

$tiny-spacing is the smallest amount of spacing between two elements. Elements separated by tiny spacing will have a strong spacial connection, such as in this heading and subheading.

Small spacing

the spacing between paragraphs.

$small-spacing is small amount of spacing between two elements. Elements separated by small spacing will have a slight spacial connection, such as between paragraphs of body text. That's why you can also use the variable $paragraph-spacing.
